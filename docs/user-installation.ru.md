# Руководство по установке для пользователей

[English version](user-installation.md)

Эта страница для пользователей, которым нужен готовый клиент FreeRDP без сборки
из исходников. Она не заменяет документацию по сборке или wiki; здесь коротко
описаны текущие варианты установки для обычных пользователей и места, где
проекту нужны сопровождающие.

## Текущие варианты установки

| Платформа | Текущий вариант для пользователя | Примечания |
| --- | --- | --- |
| Linux | [Flatpak на Flathub](https://flathub.org/en/apps/com.freerdp.FreeRDP) | Стабильная сборка для пользователей Linux. |
| Windows | [Nightly-сборки Windows](https://ci.freerdp.com/job/freerdp-nightly-windows/) | Предназначены для тестирования. Для стабильного установщика Windows нужны сопровождающие. |
| macOS | [Homebrew](https://brew.sh/) или [MacPorts](https://www.macports.org/) | Здесь пока не описан стабильный `.dmg` установщик, поддерживаемый проектом. |
| Android | Google Play или APK-канал релизов | Смотрите документацию Android и заметки к релизам. |
| iOS | App Store | Смотрите документацию iOS и заметки к релизам. |

Если для какой-либо платформы появится стабильный установщик, пожалуйста,
обновите эту страницу и ссылки для скачивания, чтобы пользователи без
технического опыта могли легко его найти.

## Подключение с macOS к компьютеру Windows

Частый домашний сценарий:

- у пользователя есть компьютер с macOS;
- пользователь хочет подключиться к компьютеру с Windows;
- оба компьютера находятся в одной сети или Windows-компьютер доступен по RDP.

В таком сценарии FreeRDP устанавливается на компьютер с macOS.
На Windows-компьютер обычно не нужно устанавливать FreeRDP; на нем должен быть
включен RDP-сервер, обычно встроенная функция Windows Remote Desktop.

Базовый чеклист:

1. В Windows включите Remote Desktop в настройках системы.
2. Убедитесь, что ваша редакция Windows поддерживает входящие подключения Remote
   Desktop. Некоторые редакции, например Windows Home, не включают встроенный
   RDP-хост.
3. Найдите имя компьютера Windows или его IP-адрес.
4. Установите FreeRDP на macOS:

   ```sh
   brew install freerdp
   ```

   или через MacPorts:

   ```sh
   sudo port install FreeRDP
   ```

5. Подключитесь с macOS. В зависимости от пакета и включенных клиентов используйте
   SDL-клиент (`sdl3-freerdp` или `sdl2-freerdp`) или `xfreerdp`:

   ```sh
   sdl3-freerdp /v:windows-hostname-or-ip /u:username
   ```

   ```sh
   sdl2-freerdp /v:windows-hostname-or-ip /u:username
   ```

   ```sh
   xfreerdp /v:windows-hostname-or-ip /u:username
   ```

По возможности вводите пароль интерактивно, а не сохраняйте его в истории shell.

## Нужна помощь: установщики для Windows и macOS

Установщики Windows `.exe` или `.msi` и пакеты macOS `.dmg` - это не разовые
файлы сборки. Для них нужны люди, которые смогут поддерживать их со временем.

Полезная работа сопровождающего:

- собирать релизные пакеты для целевой платформы;
- тестировать новые релизы на реальных системах Windows или macOS;
- поддерживать обновления установщика и изменения зависимостей;
- заниматься code signing и, для macOS, notarization;
- документировать установку и первый запуск;
- отвечать на ошибки, связанные с упаковкой и установкой.

Если вы можете помочь поддерживать стабильный способ установки для Windows или
macOS, пожалуйста, откройте issue или pull request и согласуйте работу с
сопровождающими проекта.
